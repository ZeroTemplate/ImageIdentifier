Image Identifier

This is a simple application that matches the known headers of files with the known human readable ones (JPG/BMP ... and so on). For now only 2 of those are implemented (JPG and BMP) in the next iterations new ones will be added.

How it works

Each image has it's own class, using the factory method for linking the file with the needed class. (needed for future expanding of the requirements) The logic of finding and creating each object for the files is located in the factory fileFactory class.

For example the BMP and JPG headers are stored like this:

std::vector<std::uint8_t> BMPBytes = {0x42, 0x4D}; std::vector<std::uint8_t> JPGBytes = {0xFF, 0xD8, 0xFF};

For expanding the existing code, there are 2 steps:

  • creating a new class file<imageType> and inherit the existing imageFile class.
  • extend the existing factory class to include the new image type :
    • by incrementing the number of existing IMG_OBJ_CODED
    • by adding the new type into the existing enum IMG_TYPE
    • by adding the new header into the constructor:

fileFactory() { std::vector<std::uint8_t> BMPBytes = {0x42, 0x4D}; std::vector<std::uint8_t> JPGBytes = {0xFF, 0xD8, 0xFF}; Register(IMG_TYPE::BMP, &fileBMP::Create, BMPBytes); Register(IMG_TYPE::JPG, &fileJPG::Create, JPGBytes); }
